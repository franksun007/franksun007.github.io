EXPERIENCE

Intern at Flickr during Summer 2015

The summer I spent at Flickr was one of the best summer I've ever experienced. It was my honor to work with the Flickr backend team and two smart engineers, Shawn Cook and Bhautik Joshi, on three projects.

The first one is using docker, Redis and Consul to build an automated fail-over system. It was a DevOps projects, but I majorly worked on writing the docker image for Redis and Sentinel. I also participated on writing the integrated distributed test script to test the functionality of the whole system and test the availability of Ephemeral Lock for Consul.

The second one is generating the Sitemaps for Flickr using Hadoop, Oozie and Pig Latin. The project uses Map-Reduce to manipulate the data on HDFS, concatenate them into links and publish to Google/Bing.

Last one is an Aggregation Framework, a variation for Lambda Architecture. The idea of Lambda Architecture is fusing the results of both Real-time Computing results and Batching results. I worked on the implementation of Real-time Computing component, specifically the impl of the spouts and bolts for Storm.

Undergrad Research Assistant at

I was once an Undergrad Research Assistant at UW Computer Architecture Lab - Sampa (historically meant to stand for safe multi-processing architectures) working with Brandon Holt, an amazing PhD student.

The project I worked on intended to implement the Phase Reconciliation feature to some open sourced database such as Redis Cluster. However, it was hard to find a good candidates for implementing Phase Reconciliation in all aspects - performance, threading and persistence (CAP). For example, Redis Cluster will be a good candidate, but all instances are single threading, which means the cost of reconciliation process will be much higher as all the data go through Networks comparing to multi-threading databases. The project then turns to a benchmarking task aimed to find a good live candidate to implement Phase Reconciliation.

The project used YCSB as the major benchmark tool. We benchmarked Redis Cluster, Cassandra and HyperDex. I wrote a blog post about my research experience and result. You can find it here.
